# 🌳 Day 75 – Binary Tree Traversal & DFS Patterns

## Problems Solved

1. Find Bottom Left Tree Value
2. Maximum Difference Between Node and Ancestor
3. Longest ZigZag Path in a Binary Tree
4. Vertical Order Traversal of a Binary Tree

## Key Concepts

- DFS

- Recursion
- Path-Based State Tracking
- Min/Max Tracking
- Direction Tracking
- Vertical Traversal
- TreeMap
- Priority Queue

---

## Key Learning

Today's problems covered several important binary tree patterns. The problems involved finding values at the deepest level, tracking minimum and maximum values along a path, maintaining direction-based state during DFS, and organizing nodes according to their vertical positions.

A major takeaway was understanding how additional state such as depth, minimum/maximum values, direction, row, and column can be carried through a tree traversal to solve more complex problems.

---

## Patterns Practiced

- DFS with Path State
- Direction-Based Recursion
- Coordinate-Based Tree Traversal
- TreeMap for Ordered Columns
- Priority Queue for Ordering Nodes

## Complexity

| Problem | Time | Space |
|---|---|---|
| Find Bottom Left Tree Value | O(n) | O(h) |
| Maximum Difference Between Node and Ancestor | O(n) | O(h) |
| Longest ZigZag Path in a Binary Tree | O(n) | O(h) |
| Vertical Order Traversal | O(n log n) | O(n) |

> `h` = height of the tree.

---

## Takeaway

Today's problems strengthened my understanding of how carrying additional state during tree traversal can solve complex problems efficiently. From tracking path extremes and directions to assigning coordinates to nodes, each problem introduced a different way of extending standard DFS and BFS patterns.
